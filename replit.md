# Workspace

## Overview

pnpm workspace monorepo using TypeScript. The primary user-facing app is **Yappy Video Downloader**, a no-login web utility that accepts yappy.media and rutube.ru video page URLs, calls the shared Express API, extracts direct media or stream URLs from remote page HTML/player data, and returns preview/download metadata with quality options.

## Stack

- **Monorepo tool**: pnpm workspaces
- **Node.js version**: 24
- **Package manager**: pnpm
- **TypeScript version**: 5.9
- **Frontend**: React + Vite + Tailwind CSS
- **API framework**: Express 5
- **HTTP client**: Axios in the API server
- **Video conversion**: ffmpeg for signed HLS-to-MP4 download conversion
- **Validation**: Zod (`zod/v4`), generated from OpenAPI
- **API codegen**: Orval (from OpenAPI spec)
- **Build**: esbuild for API, Vite for frontend

## Artifacts

- `artifacts/yappy-video-downloader` — React/Vite frontend at `/`
- `artifacts/api-server` — Express backend at `/api`
- `artifacts/mockup-sandbox` — canvas/component preview sandbox at `/__mockup`

## Yappy Video Downloader

### User Flow

1. User pastes a yappy.media or rutube.ru video page URL.
2. Frontend detects and displays the platform label, then posts the URL to `POST /api/extract`.
3. Backend validates the host and chooses the platform parser.
4. Yappy parser extracts media candidates from meta tags, video/source tags, JSON script data, and embedded media URLs.
5. Rutube parser extracts video ID/player data, calls Rutube player/video APIs when possible, parses direct MP4 and HLS m3u8 sources, and expands master playlists into quality options.
6. Backend returns platform, best source, selected download URL, thumbnail/title when available, and a `qualities` array.
7. Frontend shows platform, thumbnail/preview, quality selector, one download button, copy buttons, and dark/light toggle.

### API Contract

The OpenAPI spec is in `lib/api-spec/openapi.yaml` and includes:

- `GET /api/healthz`
- `POST /api/extract`

The API server also exposes `GET /api/convert` for signed HLS-to-MP4 downloads generated by `/api/extract`.

Run codegen after editing the spec:

```bash
pnpm --filter @workspace/api-spec run codegen
```

## Key Commands

- `pnpm --filter @workspace/api-server run dev` — run API server locally
- `pnpm --filter @workspace/yappy-video-downloader run dev` — run frontend locally
- `pnpm --filter @workspace/api-spec run codegen` — regenerate API hooks and Zod schemas from OpenAPI spec
- `pnpm --filter @workspace/api-server run build` — build API server
- `pnpm --filter @workspace/yappy-video-downloader run build` — build frontend

## Notes

- The downloader does not bypass private, authenticated, or DRM-protected content. It extracts media URLs that are present in public page data, player config, or public stream playlists.
- The backend intentionally rejects hosts other than yappy.media and rutube.ru.
- No database is required for this app.
